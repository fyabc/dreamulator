# 学科知识库

本目录存放各学科的参考文档，供 LLM 和开发者查阅，为世界推演提供科学依据。

## 目录结构

```
knowledge/
├── astrophysics/        # 天体物理（恒星分类、轨道力学、恒星演化）
├── planetary_science/   # 行星科学（行星形成、潮汐加热、磁场）
├── geology/             # 地质学（板块构造、火山、地貌）
├── climatology/         # 气候学（大气环流、洋流、气候分类）
├── ecology/             # 生态学（生物群落、食物网、种群动态）
└── sociology/           # 社会学（聚落模式、政治制度、文化演化）
```

## 文档规范

- 使用 Markdown 格式
- 文件名使用 snake_case，如 `orbital_mechanics.md`
- 每个学科目录包含 `CLAUDE.md` 索引文件，列出关键概念和交叉引用
- 文档应包含：
  - 核心概念和定义
  - 关键公式和参数范围
  - 现实世界的参考数据（如地球、太阳系）
  - 与其他学科的关联（如气候依赖轨道参数）

## 文档用途

1. **LLM 参考**：编写世界 input 文件时，LLM 应参考这些文档确保物理合理性
2. **引擎开发**：实现模拟引擎时，开发者参考文档中的公式和参数
3. **知识积累**：随着项目推进，持续补充和完善学科知识

## 知识积累准则（重要）

知识库是项目科学严谨性的基石。在新增或修改知识文档时，**必须**遵守以下准则：

### 1. 必须附带参考来源

- 所有新增的科学知识（公式、参数、数据、理论描述）**必须**附带可验证的参考链接或出处
- 优先使用权威来源：学术教材、NASA/USGS 等机构资料、同行评审论文
- 参考链接格式：在相关内容后标注来源，或在文档末尾的 `## 参考资料` 章节集中列出
- **没有来源的知识不得写入文档**

### 2. 严禁幻觉

- 如果对某个知识点**不确定**，必须明确承认，不得编造或猜测
- 正确的做法：诚实说明「此处我不确定，需要查证」，然后使用搜索工具获取可靠信息
- 错误的做法：凭印象写出「大概是这样」的公式或数据，然后不加标注地写入文档
- 知识库中一个错误的公式，会导致后续所有世界推演产生连锁错误——**宁缺毋滥**

### 3. 主动搜索验证

- 新增知识前，应主动使用 WebSearch/WebFetch 等工具搜索相关内容进行验证
- 即使对某个知识点有信心，也建议搜索交叉验证，确保数据时效性和准确性
- 搜索结果应作为参考来源记录在文档中

## 示例：天体物理文档结构

```markdown
# 轨道力学

## 开普勒定律

1. 行星轨道为椭圆，恒星位于焦点
2. 行星与恒星的连线在相等时间内扫过相等面积
3. 轨道周期的平方与半长轴的立方成正比

## 关键公式

### 轨道周期
T = 2π √(a³/GM)

其中：
- T: 轨道周期（秒）
- a: 半长轴（米）
- G: 引力常数
- M: 中心天体质量（千克）

### 宜居带估算
r_hz = √(L/L_sun) AU

其中 L 为恒星光度。

## 参数范围

- 偏心率 e: 0（圆）到接近 1（极扁椭圆）
- 倾角 i: 0° 到 180°
- 地球偏心率: 0.017

## 相关文档

- knowledge/planetary_science/tidal_heating.md: 偏心率影响潮汐加热
- knowledge/climatology/atmospheric_circulation.md: 轨道参数影响季节
```
