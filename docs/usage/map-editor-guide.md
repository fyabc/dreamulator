# 地图编辑器使用指南

本文档介绍 dreamulator 地图编辑器的使用方法。关于架构设计和实现细节，请参阅 [map-system.md](map-system.md)。

## 快速入门

### 1. 进入地图编辑器

从世界详情页的概览 tab 中，找到 **🗺️ 地图** 卡片：

- **已有地图**：点击地形缩略图 → 打开全页地图编辑器
- **还没有地图**：点击「生成第一张地图 →」→ 进入编辑器后点击「🌍 生成地形」

也可以直接在地址栏输入 `/worlds/<世界名>/map/<行星ID>` 进入。

### 2. 生成地形

点击左侧面板底部的 **「🌍 生成地形」** 按钮，后端会程序化生成一套完整的地形数据，包括：

- 栅格高度图（2048×1024 像素）
- Voronoi 语义网格（~1000 个单元格）
- 构造板块分配

> **当前版本**使用固定参数（3 块大陆、0.5 山脉度、15 个板块）。可调节参数的生成将在后续版本中提供。

每次点击都会重新生成整个地形。可以多试几次，直到获得满意的海陆分布。

### 3. 探索地图

生成地形后，你可以在地图上自由探索：

- **平移**：按住鼠标左键拖拽
- **缩放**：滚动鼠标滚轮
- **查看坐标**：移动鼠标，底部状态栏实时显示经纬度和海拔

### 左面板

分为两个区域：

**图层控制**（始终可用）：
- 着色模式选择
- 矢量叠加层开关

**编辑工具**（标记为「即将推出」）：
- 画笔工具
- 生成/保存按钮

### 中央地图视图

主视图区域，使用 Three.js (WebGPU/WebGL) 渲染地形，SVG 层叠加显示矢量数据。支持平移和缩放操作。水平方向支持无缝环绕（圆柱投影），可无限向左/向右拖动。

### 右面板

分为两个区域：

**单元格详情**（上部，可滚动）：
显示鼠标悬停位置的 Voronoi 单元格详细信息，包括经纬度、海拔、湿度、所属板块等属性。需要先开启「Voronoi 网格」叠加层才能悬停查看。

**鸟瞰图**（底部固定）：
缩略地图显示当前视口在全图中的位置，白色矩形标注可视范围。视口跨越地图边缘时矩形自动拆分环绕。

### 底栏

实时显示鼠标位置的坐标信息：经度、纬度、海拔（米）、像素坐标、缩放级别。

## 已实现功能

### 导航操作

| 操作 | 方式 |
|------|------|
| 平移 | 按住鼠标左键拖拽（水平无限环绕，垂直限制在地图边界内） |
| 缩放 | 鼠标滚轮（最小 = 地图填满视口，最大 20x） |
| 查看坐标 | 移动鼠标 |

### 着色模式

左侧面板的「着色模式」区域提供 4 种地形可视化方式：

| 模式 | 说明 | 用途 |
|------|------|------|
| **地形** | 海拔着色 + 山体阴影，绿色低地 → 棕色高地 → 白色山顶，蓝色海洋 | 整体地貌概览 |
| **海拔** | 灰度梯度，黑（低）→ 白（高） | 分析高程分布 |
| **海陆** | 二值着色，深色海洋 + 浅色陆地 | 查看海陆比例和海岸线 |
| **坡度** | 坡度梯度着色，平坦（蓝）→ 缓坡（绿）→ 陡坡（红） | 识别山脉和陡峭区域 |

### 矢量叠加层

左侧面板的「矢量叠加」区域可开关三种叠加层：

**Voronoi 网格**
- 以圆圈形式显示 Voronoi 单元格中心点
- 开启后，鼠标悬停可高亮单元格并在右侧面板查看属性
- 点击可选中单元格（Shift+点击多选）

**板块边界**
- 红色线段显示不同构造板块之间的交界
- 板块内的单元格会显示半透明的板块着色

**河流 / 山脊**
- 蓝色线条显示河流走向
- 橙色线条显示山脊线
- 这些数据从高度图自动提取（需要后端支持特征提取）

### 单元格详情

右侧面板显示悬停位置 Voronoi 单元格的属性：

| 属性 | 说明 |
|------|------|
| Cell #ID | 单元格编号 |
| 陆地/海洋 | 根据海拔和海平面判断 |
| 经度/纬度 | 地理坐标（度） |
| 海拔 | 以米为单位（基于 map.yaml 中的 elevation_min_m / elevation_max_m） |
| 湿度 | 当前为生成时的随机值，后续气候引擎将计算真实湿度 |
| 邻居 | 相邻单元格数量 |
| 板块 | 所属构造板块的名称和类型（oceanic/continental/mixed） |
| 生态 | 生态群系（需要生态引擎计算，当前为空） |
| 省份 | 省份 ID（文明层功能，尚未实现） |

### 程序化生成

点击左侧面板的「🌍 生成地形」按钮触发程序化生成。当前使用以下固定参数：

- **大陆数**：3
- **山脉度**：0.5（0 = 平坦，1 = 多山）
- **板块数**：15
- **地图尺寸**：2048×1024 像素
- **Voronoi 单元格数**：取决于行星参数

生成过程会自动创建高度图、Voronoi 网络和板块分配。每次点击都会用不同的随机种子重新生成。

### 保存

编辑地形后（当前版本编辑功能尚未实现），点击「💾 保存修改」将高度图上传到后端。保存按钮只在有未保存修改时可用。

## 即将推出的功能

以下功能的 UI 控件已经存在，但尚未接入后端逻辑。它们以半透明样式显示并标记为「即将推出」。

### 画笔绘制工具

计划支持 5 种画笔工具：

| 工具 | 图标 | 功能 |
|------|------|------|
| 升起 | 🔺 | 抬升画笔范围内的地形 |
| 降低 | 🔻 | 降低画笔范围内的地形 |
| 平滑 | 〰️ | 平滑画笔范围内的高程变化 |
| 平坦 | ━ | 将画笔范围内的地形拉平到统一高度 |
| 选择 | 🖱️ | 点选 Voronoi 单元格 |

画笔设置（大小、强度、硬度）也将在画笔功能实现后生效。

### 生成参数调节

计划允许用户自定义生成参数：
- 大陆数量（1–10）
- 山脉度（0–1）
- 板块数量（1–50）
- 地图分辨率

### 选择工具操作

当前可以选中 Voronoi 单元格，但尚无后续操作。计划中的功能：
- 将选中的单元格分组为省份或国家
- 批量修改选中单元格的属性
- 基于选中区域执行地形操作

## 数据存储

地图数据遵循 `input/` + `derived/` 分离原则，存储在世界的层级目录中：

```
data/worlds/<世界名>/
├── layers/geological/input/maps/<行星ID>/
│   ├── map.yaml          # 元数据（投影、尺寸、海平面等）
│   ├── elevation.png     # 16-bit 高度图（git-lfs 管理）
│   ├── voronoi.json      # Voronoi 网络（单元格、邻居关系）
│   ├── plates.json       # 构造板块（名称、类型、包含的单元格）
│   └── features.json     # 线性特征（河流、山脊、海岸线）
└── layers/geological/derived/maps/<行星ID>/
    └── terrain.png       # 地形分类图（引擎计算）
```

分支的地图数据存储在 `branches/<分支名>/layers/geological/input/maps/` 下，通过 `LayerResolver` 实现继承。
