## 1. 工具分工

Dreamulator 的地图制作采用**"外部设计 + 内部处理"**模式：

| Dreamulator 负责 | Gaea 负责 |
|---|---|
| 高度图渲染 + hillshading | 地形设计与侵蚀模拟 |
| 构造板块叠加显示 | 山脉、河谷、海岸线雕刻 |
| Voronoi 语义网格管理 | 精细的地形控制 |
| 文明地图（CivMap）涂色 | 批量地形变体生成 |

```
┌──────────────┐
│  Gaea 设计    │  节点式地形生成 + 侵蚀模拟
│  输出 16-bit  │  水力侵蚀、风化、山脊雕刻
│  TIFF 高度图  │
└──────┬───────┘
       ↓ 导入
┌──────────────────────────────────────────────────┐
│  Dreamulator                                     │
│                                                  │
│  1. 导入高度图 → 归一化 + 分辨率重采样             │
│  2. 自动 Voronoi 采样 + 板块分配                   │
│  3. 编辑器可视化: 地形渲染 + hillshading            │
│  4. 叠加 Voronoi 网格 / 板块边界 / 河流山脊        │
│  5. 在 CivMap 中为国家/省份涂色（即将推出）         │
└──────────────────────────────────────────────────┘
```

> **程序化生成作为快速原型**：Dreamulator 内置的程序化地形生成器（`terrain_generator.py`）适合快速原型——点击「🌍 生成地形」即可获得一块可用的海陆分布。但追求逼真侵蚀纹理和精确地形控制时，推荐使用 Gaea。

## 2. Gaea 项目配置

### 2.1 新建项目

1. 打开 [Gaea](https://quadspinner.com/)，创建新项目
2. 设置输出分辨率：**2048×1024**（2:1 宽高比，匹配 Dreamulator 的等距矩形投影）
   - 免费版只支持正方形输出时，可设为 **1024×1024**（导入时自动重采样到 2:1）
   - 付费版推荐 **4096×2048**，保留更多细节

### 2.2 等距矩形投影注意事项

Dreamulator 使用**等距矩形投影**（equirectangular projection），地图的横轴映射经度（-180° ~ +180°），纵轴映射纬度（90°N ~ 90°S）：

- 地图**左右边界在 ±180° 经线处相连**（太平洋中部）
- 地图**上下边界是南北极点**
- 设计地形时，左右边缘必须**无缝衔接**（详见[第 7 节](#7-接缝处理)）
- 上下边缘（极地）可以任意设计

## 3. 推荐节点图

以下是一个经过验证的通用节点图结构：

```
┌─────────────────────────────────────────────────────────┐
│  基础地形                                                │
│  ┌──────────┐     ┌──────────────┐                      │
│  │  Perlin  │────▶│  SatMask     │  控制大陆/海洋分布    │
│  │  Noise   │     │  (大陆遮罩)   │                      │
│  └──────────┘     └──────┬───────┘                      │
│                          │                              │
│  ┌──────────┐            ▼                              │
│  │  Ridge   │──▶ 叠加 ──▶ 基础高度                       │
│  │  Noise   │    (板块碰撞山系)                           │
│  └──────────┘                                           │
├─────────────────────────────────────────────────────────┤
│  侵蚀模拟                                                │
│  ┌────────────────┐    ┌──────────────────┐             │
│  │  Hydraulic     │───▶│  Thermal         │             │
│  │  Erosion       │    │  Weathering      │             │
│  │  (水力侵蚀)     │    │  (风化/热侵蚀)    │             │
│  └────────────────┘    └──────────────────┘             │
├─────────────────────────────────────────────────────────┤
│  后处理                                                  │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐        │
│  │  Terrace │────▶│  Clamp   │────▶│  Output  │        │
│  │  (阶地)   │     │  (裁剪)  │     │  (TIFF)  │        │
│  └──────────┘     └──────────┘     └──────────┘        │
└─────────────────────────────────────────────────────────┘
```

**各节点说明**：

| 节点 | 作用 | 参数建议 |
|------|------|---------|
| **Perlin/Simplex Noise** | 基础地形起伏 | 频率 2~4 控制大陆数量 |
| **SatMask / SatMap** | 大陆/海洋分布遮罩 | 调整阈值控制陆地比例 |
| **Ridge Noise** | 模拟板块碰撞形成的线性山系 | 叠加权重 0.3~0.5 |
| **Hydraulic Erosion** | 水力侵蚀，形成河谷和冲积扇 | 迭代次数 20000~50000 |
| **Thermal Weathering** | 风化作用，软化山脊 | 强度 0.3~0.6 |
| **Terrace** | （可选）形成阶地/台地效果 | 视风格需要 |
| **Clamp / Normalize** | 确保输出在 [0, 1] 范围内 | 必须添加 |

## 4. 导出高度图

在 Gaea 的 Output 节点中：

1. **格式**：选择 **16-bit TIFF**（推荐）或 32-bit float TIFF
   - 16-bit TIFF 文件较小，精度足够（65536 级高度）
   - 32-bit float TIFF 是 Gaea 的默认输出，精度更高
   - Dreamulator 两种格式都支持
2. **色彩通道**：单通道（grayscale）即可
3. **文件命名**：建议包含世界名和行星 ID，如 `myworld_planet_01_heightmap.tif`

### CLI 批量渲染（进阶）

如果有多个世界或需要批量迭代，可以使用 Gaea 的命令行渲染器：

```bash
# Windows
"C:\Program Files\QuadSpinner\Gaea\Gaea.Build.exe" ^
  --project "C:\path\to\template.gaea" ^
  --output "C:\path\to\output\myworld_heightmap.tif" ^
  --resolution 2048x1024
```

> **注意**：CLI 渲染需要 Indie 或 Pro 许可证。免费版仅支持 GUI 渲染。

## 5. 导入 Dreamulator

### 5.1 支持格式

| 格式 | 位深度 | 说明 |
|------|--------|------|
| **PNG** | 16-bit（单通道灰度） | 通用格式，所有工具都能导出 |
| **TIFF** | 16-bit（整数） | 无损，文件略大于 PNG |
| **TIFF** | 32-bit float | Gaea 原生输出，精度最高 |

> Dreamulator 内部统一转换为 `[0, 1]` 范围的 float64 数组，然后以 16-bit PNG 存储。32-bit float TIFF 的额外精度在转换过程中会被截断。

### 5.2 导入步骤

**方式 A：前端导入（推荐）**

1. 启动服务器：`uv run dreamulator serve --open`
2. 进入目标世界 → 打开**地图编辑器**
3. 在左侧面板的「数据操作」区域，点击 **「📥 导入高度图」**
4. 选择 `.png`、`.tif` 或 `.tiff` 文件
5. 等待导入完成（通常需要几秒）

**方式 B：API 导入**

```bash
curl -X POST "http://localhost:8000/api/worlds/myworld/maps/planet_01/import-elevation" \
  -F "file=@myworld_heightmap.tif"
```

### 5.3 导入后自动处理

1. **格式解码**：根据文件头自动检测格式（PNG/TIFF/BigTIFF），解码为 numpy 数组
2. **归一化**：将高度值映射到 `[0, 1]` 范围
3. **分辨率重采样**（如需要）：如果输入分辨率与项目设定不匹配（默认 2048×1024），通过双线性插值自动重采样
4. **Voronoi 重采样**：在高度图上重新生成 Voronoi 语义网格（~5000 个单元格）
5. **板块重分配**：为新网格重新分配构造板块
6. **图层注册表更新**：标记下游图层为 stale

### 5.4 分辨率建议

| 输入分辨率 | 效果 |
|-----------|------|
| 512×256 | 可用但细节模糊（适合概念验证） |
| 1024×512 | 基本够用（Gaea 免费版等效） |
| **2048×1024** | **推荐**（匹配默认项目设定） |
| 4096×2048 | 细节最丰富（会被重采样到 2048×1024） |

> 输入分辨率不需要精确匹配——Dreamulator 会自动重采样。但宽高比应为 **2:1**，否则地图会被拉伸。正方形输出（如 1024×1024）导入后会被压缩到 2:1，极地地区会显得扁平。

### 6.3 着色模式

左侧面板提供 4 种地形可视化方式：

| 模式 | 说明 | 用途 |
|------|------|------|
| **地形** | 海拔着色 + 山体阴影 | 整体地貌概览 |
| **海拔** | 灰度梯度，黑（低）→ 白（高） | 分析高程分布 |
| **海陆** | 二值着色，深色海洋 + 浅色陆地 | 查看海陆比例 |
| **坡度** | 平坦（蓝）→ 缓坡（绿）→ 陡坡（红） | 识别陡峭区域 |

### 6.4 矢量叠加层

左侧面板可开关三种叠加层：

| 叠加层 | 显示内容 |
|--------|---------|
| **Voronoi 网格** | 圆圈显示 cell 中心点，悬停高亮，Shift+点击多选 |
| **板块边界** | 红色线段显示不同板块交界 |
| **河流 / 山脊** | 蓝色线条 = 河流，橙色线条 = 山脊线 |

### 6.5 单元格详情

开启 Voronoi 网格叠加后，鼠标悬停可在右面板查看属性：

| 属性 | 说明 |
|------|------|
| Cell #ID | 单元格编号 |
| 陆地/海洋 | 根据海拔和海平面判断 |
| 经度/纬度 | 地理坐标（度） |
| 海拔 | 米（基于 map.yaml 中的 elevation_min_m / elevation_max_m） |
| 板块 | 所属构造板块名称和类型 |

### 6.6 程序化生成（快速原型）

如果行星还没有地图数据且不想使用外部工具，编辑器中央的「🌍 生成第一张地图」按钮会使用内置程序化生成器创建基础地形：

```bash
uv run dreamulator build <世界名>  # 通过 CLI 自定义参数生成
```

## 7. 接缝处理

### 问题

Dreamulator 使用等距矩形投影，地图的左右边界在 ±180° 经线处相连。如果高度图的左右边缘高度值不一致，在 3D 球体渲染或地图平移时会出现明显的**接缝**（垂直断裂线）。

### 在 Gaea 中预防

- 在噪声节点中使用 **tileable/wrapping** 选项（如果可用）
- 避免在 ±180° 经线附近放置大型地形特征——将这块区域设计为开阔海洋是最安全的做法

### 在 Photoshop/GIMP 中修复

1. 打开高度图
2. 使用"位移"滤镜（Filter → Other → Offset）水平移动 50%
3. 在现在位于中间的原始接缝处使用修复画笔/克隆图章工具
4. 移回原位并保存

### Dreamulator 的处理

- **程序化生成**：自动使用余弦插值融合左右边缘（`terrain_generator.py` 中的 `_blend_horizontal_wrap()`）
- **外部导入**：**不会**自动处理接缝，需在外部工具中修复
- **Voronoi 网格**：已在 ±360° 处放置幽灵点，即使高度图有轻微接缝，语义网格的板块分配和单元格邻接关系仍然正确

## 9. 附录

### Gaea 版本对比

| 版本 | 价格 | 分辨率上限 | 商用许可 |
|------|------|-----------|---------|
| **Free** | $0 | 1024² | ❌ 非商用 |
| **Indie** | $99（永久） | 4K | ✅ 年收入 < $100K |
| **Pro** | $199（永久） | 8K+ | ✅ 无限制 |

> 对于大多数项目，免费版（1024² 非商用）已经足够。导入时会自动重采样，1024×1024 的输出效果完全可以接受。

### 多精度地图

Dreamulator 采用**单一真相源**策略：

| 类型 | 分辨率 | 用途 |
|------|--------|------|
| **Master** | 最高 (4096×2048) | 唯一编辑/导入对象，所有变体从此派生 |
| **Overview** | 较低 (1024×512) | 快速浏览和全局推演 |
| **Display** | 中等 (2048×1024) | 展示和视频制作 |
| **Regional** | 区域裁切 | 局部推演，可叠加高频细节 |

编辑 Master 后，变体自动标记为需要重新生成。

### 常见问题

**Q: 导入的 TIFF 分辨率与项目不同怎么办？**
A: Dreamulator 会自动重采样。降采样使用 LANCZOS 算法，上采样使用 BILINEAR。

**Q: 为什么我的地形导入后看起来不一样？**
A: Dreamulator 将高度值归一化到 [0, 1] 范围。如果 TIFF 值域不是标准范围，会自动缩放。

**Q: Earth 分支的 CivMap 受影响吗？**
A: 不受影响。Earth 分支使用真实地球 GeoJSON（Natural Earth 数据），不依赖 Voronoi 系统。

### 数据存储

地图数据存储在世界的层级目录中：

```
data/worlds/<世界名>/
├── layers/geological/input/maps/<行星ID>/
│   ├── map.yaml          # 元数据（投影、尺寸、海平面等）
│   ├── elevation.png     # 16-bit 高度图（git-lfs 管理）
│   ├── voronoi.json      # Voronoi 网络
│   ├── plates.json       # 构造板块
│   └── features.json     # 线性特征（河流、山脊）
└── layers/geological/derived/maps/<行星ID>/
    └── terrain.png       # 地形分类图（引擎计算）
```

分支的地图数据存储在 `branches/<分支名>/layers/geological/input/maps/` 下，通过 `LayerResolver` 实现继承。
